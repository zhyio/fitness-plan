# 多账号管理与数据隔离功能测试指南

本文档旨在提供针对应用最新加入的 **Supabase Auth 鉴权系统** 以及 **IndexedDB 离线数据隔离** 功能的完整黑盒测试步骤。

## 测试前提
- 拥有可以访问外网/连接至 Supabase 数据库的网络环境。
- 确保您的 Supabase 后台 (`Authentication -> Providers -> Email`) 已将 **`Confirm email`** 选项**关闭**（因为我们前端使用了账号转换为虚拟邮箱的方案进行免邮箱验证登录）。
- 使用任意现代浏览器（Chrome、Safari 等），可以使用无痕模式 (Incognito) 来避免受到历史缓存影响。

---

## 阶段一：纯净启动与未登录拦截测试

**测试目标：** 确保未登录状态下，无法看到主应用界面且被强制定向至 Auth 面板。

1. **环境准备**：在浏览器中打开应用链接（或本地 `index.html`）。如果之前有登录状态，请先在“动作设置”中点击“退出登录”。
2. **UI 验证**：
   - [ ] 页面背景呈现深色。
   - [ ] 屏幕正中仅显示“Fitness Plan 记录每一次泵感”字样以及账号、密码输入框，以及“去注册”和“登录”按钮。
   - [ ] 无法滑动看到任何顶栏标题、日历按钮，以及底部的导航 Tab。
   - [ ] 查看浏览器开发者工具的 Console 控制台，确认没有向 Supabase 数据表发起越权的 SELECT 请求。

---

## 阶段二：注册与登录交互测试

**测试目标：** 验证虚拟邮箱拼接机制以及注册浮层的展现与工作逻辑。

1. **打开注册弹窗**：
   - 点击“去注册”按钮。
   - [ ] **视觉检查**：弹出带标题为“注册新账号”的对话框（Modal）。此弹窗必须位于最顶层，没有半透明的虚影重叠。
2. **表单非空与长度限制验证**：
   - 留空点击“确认注册”，提示“请输入账号和密码”。
   - 输入密码 `123`，提示“密码至少需要 6 位”。
3. **完成注册**：
   - 输入测试账号（如：`testUser1`）和测试密码（如：`123456`）。
   - [ ] 点击“确认注册”，等待 1-2 秒，提示“注册成功！”。
   - [ ] 弹窗自动关闭，并且主界面的登录账号和密码框自动回填为刚刚输入的内容。
4. **进行登录**：
   - [ ] 点击“登录”按钮。
   - [ ] 登录成功后，黑色 Auth 页面瞬间隐去，主界面（骨架屏及 Tab）呈现，进入胸部（Chest）训练看板。

---

## 阶段三：云端数据保存与隔离测试

**测试目标：** 验证数据是否被正确写入并绑定到专属的 UID，且不会发生本地（IndexedDB）或云端串号。

### 子步骤 3A: 数据录入 (测试账号 A)
1. 在刚刚登录的 `testUser1` 账号下：
   - [ ] 进入“胸部” Tab，点击悬浮的 `+` 按钮。
   - [ ] 添加一个新动作（例如：“杠铃卧推”，组数 4，次数 10，重量 60kg）。
   - [ ] 点击打卡点亮 2 个组数的圆环。
2. **强制刷新**：
   - [ ] 刷新页面，您不需要重新登录，应用无缝直接进入主界面。
   - [ ] “杠铃卧推”依然在页面上，且那 2 组打卡状态依然被保留。

### 子步骤 3B: IndexedDB 本地隔离擦除验证
1. [ ] 点击顶部的“动作设置 (齿轮图标)”。
2. [ ] 滑动到底部，点击红色的“退出登录”。
3. [ ] 页面瞬间跳回至纯粹的登录 Auth 面板。
4. 打开浏览器开发者工具 (`F12`) -> `Application` (应用) -> `IndexedDB` -> `fitness-plan-cache`。
   - [ ] 验证 `kv` 表格是空的。
   - [ ] 验证 `workout_history` 表格是空的。

### 子步骤 3C: 隔离独立性 (测试账号 B)
1. 再次点击“去注册”，使用全新的账号注册（例如：`testUser2`，密码 `123456`）。
2. 点击“登录”进入主界面。
3. **数据独立性验证**：
   - [ ] 主页面必须是**完全空白**的状态。
   - [ ] 绝对看不到之前 `testUser1` 添加的“杠铃卧推”数据。
   - [ ] 点击顶部的日历统计，历史记录完全空白。
4. (可选验证) 添加一条专属于 `testUser2` 的记录，例如“哑铃飞鸟”。再退回 `testUser1` 登录，只能看到各自的数据。

---

## 阶段四：离线平滑降级（可选高级测试）

**测试目标：** 在失去网络或无 Supabase 凭据的情况下，App 是否能提供基本的功能访问。

1. 切断设备的网络连接，强制刷新页面（需 PWA 缓存支持或本地运行）。
2. [ ] 脚本尝试加载 Supabase Client 失败或捕获错误。
3. [ ] 自动隐藏 Auth 登录面板，并分配默认 `USER_ID = 'offline_user'`。
4. [ ] 页面出现主界面并可以正常点击记录本地缓存（IndexedDB），直至下次联网恢复。
