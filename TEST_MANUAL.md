# 极简云同步健身计划 (Fitness Plan) - 全功能测试手册

本测试手册根据项目 `README.md` 中的核心功能与部署配置编写，用于全面验证应用的各项特性是否达到预期，特别注重移动端体验与无缝云同步的测试。

---

## 1. 测试环境准备

- **硬件设备**: 
  - 智能手机（主测试平台，测试触控与 PWA 表现）。
  - PC/Mac（辅助测试跨设备同步）。
- **浏览器**: Safari (iOS), Chrome (Android/PC)。
- **网络状态**: 正常网络连接、断网/离线状态。
- **访问入口**: 
  - 线上环境：[GitHub Pages 部署链接](https://zhyio.github.io/fitness-plan/)
  - 本地环境：运行 `python3 -m http.server 5179 --bind 127.0.0.1` 后打开 `http://127.0.0.1:5179/`。

## 2. 功能测试用例 (Functional Testing)

### 2.1 单用户免登与云端同步 (Private Sync)
| 用例编号 | 测试模块 | 测试操作 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-1.1** | 单用户免登 | 首次打开应用页面（清理缓存后）。 | 无需任何账号注册、密码输入流程，直接进入主界面。联网时连接 `87553652@qq.com` 的私有数据；远端不可用时进入本地模式。 |
| **TC-1.2** | 跨端实时同步 | 设备 A 增加一个新动作并打卡；保持设备 B 页面同时打开或稍后打开。 | 设备 B 无需手动刷新，能毫秒级同步（或重新加载后可见）设备 A 的新增动作和打卡状态。 |
| **TC-1.3** | 状态记忆恢复 | 切换至底部“部位分化”Tab，选择右下角“背”部，关闭浏览器后重新进入应用。 | 页面自动加载上次退出的状态：停留在“部位分化”页面，且选中的依然是“背”部。 |

## 4. 边界与异常测试 (Edge Cases & Exceptions)

| 用例编号 | 测试模块 | 测试操作 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-8.1** | 无网平滑降级 | 断开设备网络，尝试进行打卡操作，之后刷新。 | Jsdelivr CDN 资源失效时本地基础功能依旧工作；断网打卡操作不引发严重崩溃，网络恢复后应能恢复云同步。 |
| **TC-8.2** | 极端输入测试 | 在新建动作中输入极长的名字、超过 50 组的组数、极大的重量数值。 | 界面不发生明显破损变形，文本正常截断/换行，圆圈生成合理或有数值上限防护。 |
| **TC-8.3** | 空状态 UI | 刚重置或未添加任何动作时，查看主页或部位列表。 | UI 展示美观的空状态占位图或引导文案。 |
